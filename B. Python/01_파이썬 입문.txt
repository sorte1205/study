- 프로그래밍 언어의 구분 
	- 컴파일 언어
		- 컴파일: 2세대 이후의 프로그래밍 언어로 작성된 소스코드는 컴퓨터가 실행할 수 있는 기계어(이			진수)로 변환하는 과정
		- 컴파일러: 컴파일을 수행하는 소프트웨어
		- 컴파일 언어의 동작 과정
			- 소스코드 -> 컴파일러 -> 실행파일 -> 컴퓨터를 통한 실행
			- 운영체제 종류나 CPU 종류에 따라 개별 수행. (8번)

	- 인터프리터 언어
		- 인터프리터: 소스코드를 컴파일 과정 없이 바로 실행하는 프로그램 또는 환경
		- 인터프리터 언어:	인터프리터 환경에서 동작하도록 고안된 프로그래밍 언어 =스크립트 언어)
			- py, js
		- 인터프리터 언어의 동작 과정
			- 소스코드 -> 인터프리터 -> 컴퓨터를 통한 실행
			ex. 이력서	MSword		PC
			- 개발자는 별도의 컴파일 과정 없이 바로 실행 가능
			- 컴퓨터 종류에 따른 컴파일 과정이 생략, 결과 확인이 빠르다.
			- 사용자도 프로그램을 실행하기 위해서는 소스코드를 인터프리터를 통해 해석하는 과정				이 필요하다. (입사시험에 파이썬 배우기도)
